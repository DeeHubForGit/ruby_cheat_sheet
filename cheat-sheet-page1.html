Output

puts

puts "This will print the text to the screen then move to the next line"

print

print "This will print the text to the screen without moving to the next line"

Comments

  • Comments start with #
  • Used to explain code
  • Ignored when the program runs
# This is a comment
puts "This runs"

# Ruby does not have a true multi-line comment.
# Use # at the start of each line.
# All lines starting with # are comments.

Variables

Local Variables

  • Start with a lowercase letter or underscore and an underscore between multiple words (snake_case).
  • Scope is limited to the block or method where the variable is defined.
  • Do not need to be declared before use.
  • Are created when first assigned a value.
count = 10
_name = "Ruby"
total_price = count * 5

Constants

  • Start with an uppercase letter
  • Used for values that should not change
  • Ruby allows reassignment but shows a warning
MAX_ITEMS = 10
TAX_RATE = 0.1

total = MAX_ITEMS * TAX_RATE

Naming Style

  • Use snake_case for variable names.
  • Be descriptive, not abbreviated.
  • Avoid using reserved words as variable names.

Good Naming

user_name = "Joe"
total_items = 3
is_valid = true

Bad Naming

x = 5
tmp = "data"

Dynamic Typing

name = "Joe"
age = 30
puts "#{name} is #{age}"

Basic Data Types

# String: text data
"Hello, World!"

# Integer: whole numbers
42
-7

# Float: decimal numbers
3.14
-0.001

# Boolean: true or false
true
false

# Array: ordered list of items
[1, 2, 3]

# Hash: key-value pairs
{ name: "Alice", age: 25 }

# Nil: represents no value
nil

Arrays

  • Ordered list of items
  • Indexed by position (starts at 0)
  • Can create, access, add and remove items
  • << or push appends an item to the end
# Create an array
numbers = [1, 2, 3]

# Access elements (index starts at 0)
numbers[0]   # 1
numbers[1]   # 2

# Add elements
numbers << 4        # append using <<
numbers.push(5)     # append using method

# Remove elements
numbers.pop         # removes last item

# Resulting array
# [1, 2, 3, 4]

Hashes

  • Key–value data structure
  • Keys are usually symbols or strings
  • Values accessed using keys
  • Can add and remove key–value pairs
# Create a hash
# Keys can also be symbols (shown later)
user = {
  "name" => "Alice",
  "age"  => 25
}

# Access values
user["name"]   # "Alice"
user["age"]    # 25

# Add a key-value pair
user["email"] = "alice@example.com"

# Remove a key-value pair
user.delete("age")

Symbols (:symbol)

  • Lightweight, immutable (cannot be changed) identifiers.
  • Used as names or labels, not data.
  • Common as hash keys and method arguments.
# Symbol
:status
:user_id
:is_valid

"name"    # String (can be changed)
:name     # Symbol (cannot be changed)

Conditionals

if, elsif, else

# Basic if statement
if total > 40
    puts "Large order"
end

# Ruby-style one line
puts "Large order" if total > 40

# Using elsif and else
if total > 100
  puts "Free shipping"
elsif total > 40
  puts "Standard shipping"
else
  puts "Shipping applies"
end

Case / When (conditional branching)

  • Used to compare one value against many options
  • Cleaner than long if / elsif chains
  • Uses when instead of elsif
status = :paid    # value being checked

case status
when :paid
  puts "Order paid"
when :pending
  puts "Awaiting payment"
else
  puts "Unknown status"
end

Comparison & Logical Operators

  • Used to compare values
  • Used in conditionals and loops
  • Return true or false
a = 10
b = 5

a == b     # false
a != b     # true
a > b      # true
a <= b     # false

a > 5 && b < 10   # true
a < 5 || b < 10   # true
!false            # true

Blocks (do..end, { })

  • Blocks are chunks of code passed to methods and executed later.
  • Use do..end for multi-line blocks.
  • Use { } for single-line blocks.
# Multi-line block
[1, 2, 3].each do |n|
    puts n
end

# Single-line block
[1, 2, 3].each { |n| puts n }

Loops/Iteration

  • Each: Iterates over an array:
orders.each do |order|
    puts order[:id]
end
  • Times: Repeats a block a specified number of times:
5.times do |i|
    puts "Iteration #{i + 1}"
end

Common Enumerable Methods

  • Used with arrays and collections
  • Return new values, do not modify originals
  • Very common in Ruby and Rails
numbers = [1, 2, 3, 4, 5]

numbers.map { |n| n * 2 }        # [2, 4, 6, 8, 10]
numbers.select { |n| n.even? }   # [2, 4]
numbers.reject { |n| n < 3 }     # [3, 4, 5]
numbers.find { |n| n > 3 }       # 4

Methods

  • Reusable blocks of code that perform a task
  • Used to organise code and reduce repetition (DRY)
  • Can take arguments (parameters)
  • The last evaluated expression is returned automatically (implicit return)
  • return is optional but exits the method immediately
  • Parentheses are optional when calling methods with no arguments
def total(price, qty)
    price * qty
end
